  The big O notation is a mathematical notation which used to display the worst-case scenario of an algorithm. There are two types of complexities we should consider in software development process.

  1. Time Complexity
  2. Space Complexity

Time Complexity


A valid algorithm takes a finite time to execute. The time require to solve a given problem is known as the time complexity for that problem. We use the big O notation to display the time complexity of the worst-case scenario.

There are few types of time complexities, and these are few of them.

  • Constant runtime – O(1)
  • Logarithmic runtime – O(log n)
  • Linear runtime – O(n)
  • Log-Linear runtime – O(n log n)
  • Quadratic – O(n²)
  • Exponential – O(2ⁿ)
  • Worse – O(n!)


Space Complexity


Just like time complexity, space complexity is an important aspect to consider when evaluating an algorithm. Space complexity refers to the amount of memory an algorithm needs to run to completion. It includes both the memory needed for the input data and the memory needed for the algorithm’s operations.

  • Constant space – O(1)
  • Logarithmic space – O(log n)
  • Linear space – O(n)
  • Quadratic space – O(n²)
  • Exponential space – O(2ⁿ)
